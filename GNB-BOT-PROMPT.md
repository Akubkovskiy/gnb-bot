# GNB Docs Bot — Полное ТЗ для Claude Code

## Контекст
Создай Telegram-бота для автоматизации исполнительной документации
по объектам ГНБ (горизонтально-направленное бурение).
Бот работает ЛОКАЛЬНО на Windows 11 ПК.
Используется подписка Claude Max (НЕ API-ключ).
Язык общения бота: русский.

## Пользователь
Инженер ПТО. Занимается разработкой исполнительной документации
после выполнения проколов методом ГНБ. Работает в AutoCAD и Excel.

## ЧАСТЬ 1: АРХИТЕКТУРА ПРОЕКТА

### 1.1 Стек технологий

Проанализируй и выбери лучший стек для этой задачи.
Рекомендуемый:

- **Язык**: TypeScript / Node.js (или Python — выбери обоснованно)
- **Telegram Bot**: grammy / telegraf (TS) или aiogram 3.x (Python)
- **Claude Code**: вызов через `claude -p "..."` как subprocess
  с `--output-format stream-json` для стриминга ответов
- **Excel .xls**: xlrd + xlwt + xlutils (Python) или SheetJS (TS)
  ВАЖНО: шаблоны в формате .xls (старый Excel), НЕ .xlsx!
- **STT голос**: faster-whisper (локально, бесплатно, русский язык)
- **База данных**: JSON-файлы в .gnb-memory\ (подробнее ниже)
- **Автозапуск**: PM2 или Windows Task Scheduler

### 2.2 Правила создания директорий

СТРОГО СОБЛЮДАТЬ:
1. Корень: `C:\Users\kubko\YandexDisk\Работа\`
2. Уровень 1: `[Заказчик]\` — бот СПРАШИВАЕТ если не указан
3. Уровень 2: `[Объект]\` — бот СПРАШИВАЕТ если не указан
4. Если папка не существует — СПРОСИТЬ пользователя, создать ли
5. При создании показать полный путь и подтвердить

### 2.3 Именование файлов

Формат: `[Объект] [ТипАкта] [НомерГНБ].xls`
Ревизии: `[Объект] [ТипАкта] [НомерГНБ] изм[N].xls`

Примеры:
- `ЗИЛ ЗП 11-11.xls` — первая версия
- `ЗИЛ ЗП 11-11 изм1.xls` — первая правка
- `ЗИЛ АОСР 11-11.xls`
- `ЗИЛ АОСР 11-11 изм2.xls`

Типы актов:
- ЗП = Акт защитного покрытия ГНБ
- АОСР = Акт освидетельствования скрытых работ

## ЧАСТЬ 4: ШАБЛОНЫ EXCEL — КАРТА ЯЧЕЕК

### 4.1 Файл "Акты ГНБ" — 10 листов

Внутри 10 листов: Лист1, Герметизация, Акт освид, СКРЫТЫЕ РАБОТЫ,
ПКЛ, СДАЧИ-ПРИЕМКИ, СООСНОСТЬ, Сварка труб, ТН, Опись.

КЛЮЧЕВОЕ: все 10 листов ссылаются на Лист1 через формулы!
Бот заполняет ТОЛЬКО Лист1 — остальное обновляется автоматически.

Лист1 — карта ячеек для заполнения:
```
R2  C1  → Наименование объекта (КЛ ГНБ)
R4  C1  → Адрес
R6  C1  → Направление (1КЛ от РТП X до РТП Y)
R7  C12 → Заказчик (организация)
R8  C12 → Эксплуатация (организация)
R9  C12 → Генподрядчик (организация)
R10 C12 → Субподрядчик (организация)
R12 C1  → Номер перехода (например: №11-11)
R14 C1  → Номер/шифр проекта
R16 C1  → Проектировщик (организация)
R18 C1  → Дата начала работ (Excel serial number!)
R20 C1  → Дата окончания работ (Excel serial number!)
R22 C1  → Исполнитель (организация)
R22 C7  → Номер СРО исполнителя
R24 C1  → Дата завершения текстом ("«22» октября 2025 г.")
R38 C1  → Марка трубы (полная: Труба ЭЛЕКТРОПАЙП 160/120-N1250 F2 SDR 13,6)
R42     → Параметры перехода: L_план, L_профиль, кол-во труб, L_общая, d_скважины, конфигурация, стыков
```

ВАЖНО про даты:
- R18 и R20 хранят даты как Excel serial number
- Формула: serial = days_since(1900-01-01) + 1
- Пример: 12.10.2025 = 45942, 22.10.2025 = 45952
- Бот ДОЛЖЕН корректно конвертировать даты в Excel serial

### 4.2 Файл "АОСР" — 3 листа

Листы: Лист1, АОСР (1), АОСР (2).
Бот заполняет ТОЛЬКО Лист1 — остальное обновляется автоматически.

Лист1 — карта ячеек:
```
R2 C2        → Номер перехода (11-11)
R3 C2        → L профиля (86.62)
R3 C3        → Кол-во труб (2)
R4 C2        → L общая (173.24)
R5 C2        → Адрес
R6 C3        → День начала (12)
R6 C4        → Месяц начала текстом ("октября")
R6 C5        → Год начала (2025)
R7 C3        → День окончания (22)
R7 C4        → Месяц окончания ("октября")
R7 C5        → Год окончания (2025)
R8 C3        → День АОСР (22)
R8 C4        → Месяц АОСР ("октября")
R8 C5        → Год АОСР (2025)
```

ВАЖНО: даты в АОСР хранятся по-другому — день, месяц текстом, год отдельно!

## ЧАСТЬ 5: ОСНОВНЫЕ СЦЕНАРИИ

### 5.1 Создание нового комплекта актов

ПОЛНЫЙ СЦЕНАРИЙ (бот следует этому алгоритму):

```
1. Пользователь: "Новый ГНБ" или "Новый заказ"

2. Бот спрашивает (если не указано):
   - Заказчик? (ищет в базе, предлагает существующих)
   - Объект? (ищет в папках заказчика)
   - Номер ГНБ?

3. Бот проверяет/создаёт директорию:
   Работа\[Заказчик]\[Объект]\
   Если нет — спрашивает, создать ли.

4. Бот просит документы:
   "Жду документы для ГНБ №X:
    ☐ Исполнительная схема (PDF)
    ☐ Акт осмотра (фото рукописного)
    ☐ Паспорт трубы (PDF или фото)
    ☐ Образец акта (Excel прошлого ГНБ) — опционально
    Отправляй в любом порядке."

5. По мере получения документов:

   PDF исполнительной схемы → извлекает:
   - Номер ГНБ, шифр проекта (из штампа)
   - L по плану, L по профилю
   - Количество труб, диаметр, d скважины
   - Адрес (из штампа)

   Фото рукописного акта → OCR → извлекает:
   - Подписанты (ФИО, должности)
   - Даты работ
   - Дополнительные данные

   PDF/фото паспорта трубы → извлекает:
   - Полная марка трубы
   - ТУ, ГОСТ
   - Номер сертификата, срок действия
   - Производитель

   Excel образец → извлекает:
   - Все данные из Лист1
   - Подписанты, организации, СРО

6. Бот показывает ВСЕ извлечённые данные:
   "Вот что собрал из документов:
    [таблица всех полей]

    Проверяю подписантов по базе:
    ✅ Байдаков Н.Н. — приказ №287-к, действующий
    ⚠️ Литвяк Д.В. — нет в базе

    Всё верно? Что исправить?"

7. Пользователь подтверждает / вносит правки

8. Бот генерирует:
   - Копирует шаблон (образец или пустой)
   - Заполняет Лист1 (обе файла)
   - Сохраняет в папку объекта
   - Отправляет файлы в Telegram
   - Обновляет базу знаний

9. Бот показывает итог:
   "✅ Создано:
    📎 ЗИЛ ЗП 11-11.xls
    📎 ЗИЛ АОСР 11-11.xls
    📁 Сохранено: Работа\Крафт\ЗИЛ\
    🔄 Яндекс Диск синхронизирует автоматически"
```

### 5.2 Правки по замечаниям заказчика

```
1. Пользователь отправляет замечания:
   - Текстом: "В ЗП 11-11 замени длину на 87,10"
   - Скриншотом замечания от заказчика
   - Или бот сам получает email с замечаниями

2. Бот распознаёт замечания и формирует список правок

3. Бот показывает diff:
   "Изменения в ЗИЛ ЗП 11-11:
    ❌ Лист1 R42: L_профиль 86,62 → 87,10
    ❌ Лист1 R20: дата 22.10 → 23.10
    Применить?"

4. Пользователь подтверждает

5. Бот:
   - Вносит изменения
   - Сохраняет с пометкой "изм1"
   - Обновляет историю ревизий в базе
   - Отправляет файл в Telegram

6. Опционально: бот сам отправляет по email заказчику
```

### 5.3 Поиск по базе знаний

Бот отвечает на вопросы на естественном языке:

```
"Кто проверяющий на ГНБ 15?"
→ ищет в gnb-transitions.json по номеру ГНБ
→ показывает проверяющих с контактами

"Какая длина была у ГНБ 11-11?"
→ показывает L_план, L_профиль, L_общая

"Кто технадзор в 3-м районе?"
→ фильтрует people.json по роли и району

"Сколько ГНБ сделали по ЗИЛу?"
→ считает переходы в gnb-transitions.json

"Запомни телефон Гойдукова +7 925 999-88-77"
→ обновляет people.json
```

### 5.4 Добавление специалиста из приказа

```
1. Пользователь скидывает фото/PDF приказа

2. Бот OCR → извлекает:
   - ФИО
   - Должность
   - Номер приказа, дата
   - Кем выдан

3. Бот показывает:
   "Распознал приказ:
    Гойдуков А.С., Ведущий специалист ОТН
    Приказ №341-к от 15.09.2025, АО «ОЭК»
    Добавить в реестр?"

4. Пользователь подтверждает

5. Бот сохраняет скан в .gnb-memory/docs/
   и обновляет people.json
```

### 5.5 Email-цикл (через Gmail)

```
1. Бот периодически проверяет Gmail
   или пользователь говорит "проверь почту"

2. Бот: "📬 Новое письмо от kraft@company.ru:
    Тема: Замечания по ГНБ 11-11
    [содержание замечаний]
    Подготовить правки?"

3. Пользователь: "Да"

4. Бот готовит правки → показывает diff → подтверждение

5. Пользователь: "Отправь им"

6. Бот отправляет ответ с исправленным файлом
   через Gmail
```

## ЧАСТЬ 6: CLAUDE.MD — ИНСТРУКЦИИ БОТА

```markdown
# GNB Docs Bot — Персональный ассистент инженера ПТО

## Язык
ВСЕГДА отвечай на русском языке.

## Роль
Ты — ассистент инженера ПТО по ГНБ (горизонтально-направленное бурение).
Помогаешь с исполнительной документацией: акты, АОСР, реестры.

## Принципы работы
1. ВСЕГДА показывай что ты извлёк/распознал и спрашивай подтверждение
2. НИКОГДА не сохраняй файл без подтверждения пользователя
3. При неуверенности — спроси, не угадывай
4. Каждое изменение в акте — показывай diff
5. После сохранения — покажи полный путь к файлу
6. Обновляй базу знаний (.gnb-memory) после каждого действия

## Приоритет источников данных
1. Ручные правки пользователя ← ВЫСШИЙ ПРИОРИТЕТ
2. Рукописный акт приёмки (OCR)
3. PDF исполнительной схемы
4. Паспорт трубы (OCR)
5. Образец Excel (прошлый ГНБ)
6. База знаний (.gnb-memory) ← НИЖНИЙ ПРИОРИТЕТ
```

## ЧАСТЬ 7: ОБРАБОТКА ВХОДЯЩИХ ДАННЫХ

### 7.1 Типы входящих сообщений

Бот должен автоматически определять тип входящего сообщения:

| Входящее | Что делать |
|----------|-----------|
| Текст "новый ГНБ" / "новый заказ" | Запустить сценарий 5.1 |
| Текст "замени X на Y" | Запустить сценарий 5.2 |
| Текст-вопрос ("кто?", "когда?", "сколько?") | Запустить сценарий 5.3 |
| Текст "запомни..." | Обновить базу знаний |
| Текст "проверь почту" | Проверить Gmail |
| Фото (JPEG/PNG) | OCR → определить тип документа |
| PDF | Определить: схема? паспорт? приказ? акт? |
| Excel .xls | Считать как образец |
| Голосовое сообщение | STT → обработать как текст |
| Скриншот замечаний | OCR → сценарий 5.2 |

### 7.2 Автоопределение типа документа (OCR)

Когда пользователь присылает фото или PDF, бот определяет:

```
"Паспорт качества" в тексте → паспорт трубы
"Сертификат соответствия" → сертификат
"Акт осмотра" / рукописный → акт приёмки
"Приказ" / "О назначении" → приказ о назначении
Штамп чертежа → исполнительная схема
Таблица с подписями → акт (Excel-формат в PDF)
```

### 7.3 STT (голосовые сообщения)

Telegram присылает .ogg файл. Обработка:
1. Конвертировать .ogg → .wav (ffmpeg)
2. Транскрибировать через faster-whisper (модель base, язык ru)
3. Обработать текст как обычное текстовое сообщение
4. Показать транскрипцию пользователю перед обработкой
