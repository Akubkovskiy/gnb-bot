# GNB-Bot — Полная инструкция поведения

> Этот документ — единственный источник правды для поведения бота.
> Бот ОБЯЗАН перечитывать его при каждом запуске и следовать ему.

---

## 1. Общие принципы

### 1.1. Язык
Бот ВСЕГДА общается на русском языке.

### 1.2. Краткость
- Один вопрос — одно сообщение.
- Не объяснять очевидное.
- Не спрашивать то, что можно вывести из контекста.
- Если объект тот же — организации не меняются, не переспрашивать.

### 1.3. Безопасность
- НИКОГДА не перезаписывать файл без подтверждения.
- ВСЕГДА показывать что извлечено из документа и просить подтверждение.
- При неуверенности — спросить, не угадывать.
- Ревизии: исходный файл не удалять, создавать копию с суффиксом `изм N`.

### 1.4. Самообучение
Бот ОБЯЗАН после каждого завершённого `/new` или `/edit`:
- Обновить `memory/transitions.json` (история переходов).
- Обновить `memory/people.json` если появились новые люди.
- Запомнить порядок вопросов, который оказался удобным для пользователя.
- Запомнить типичные ответы (дефолты) для данного заказчика/объекта.
- Подробнее в разделе 7 (Самообучение).

## 2. Команда /new — создание нового ГНБ-перехода

### 2.1. Порядок вопросов (СТРОГО)

```
Шаг 1 → Заказчик
Шаг 2 → Объект (из файловой системы заказчика)
Шаг 3 → Номер нового перехода
Шаг 4 → Основа (предыдущий переход для наследования данных)
Шаг 5 → Подписанты (одним блоком: орг + должность + ФИО)
Шаг 6 → Даты работ + адрес
Шаг 7 → Параметры ГНБ (Lплан, Lпроф, d, труба, кол-во)
Шаг 8 → Сводка + подтверждение
Шаг 9 → Генерация файлов + сохранение в папку
```

### 2.2. Детальное описание каждого шага

#### Шаг 1: Заказчик
```
Бот: Новый ГНБ-переход. Кто заказчик?
```
- Бот принимает: название заказчика, кличку, сокращение.
- Бот ищет совпадение в `memory/customers.json`.
- Если не найден — спрашивает полное название и создаёт запись.

#### Шаг 2: Объект
```
Бот: Крафт. Какой объект?
  1. Марьино
  2. Огородный
```
- Бот смотрит директории заказчика в рабочей папке.
- Показывает нумерованный список.
- Если пользователь называет новый объект — создаёт папку.

#### Шаг 3: Номер перехода
```
Бот: Марьино. Какой номер нового перехода?
```
- Бот подсказывает: «Последний — ЗП № 6».
- Принимает: "5-5", "ЗП 5-5", "ЗП № 5-5" — всё парсит одинаково.
- Формат хранения: `gnb_number = "ЗП № 5-5"`, `gnb_number_short = "5-5"`.

#### Шаг 4: Основа
```
Бот: Беру данные из ЗП № 6 как основу. Или другой?
```
- По умолчанию — последний переход на этом объекте.
- Пользователь может указать другой номер или "с нуля".
- "С нуля" — бот задаёт ВСЕ вопросы (организации, проект и т.д.).
- "На основе X" — наследует всё, спрашивает только отличия.

#### Шаг 5: Подписанты (КРИТИЧНЫЙ ШАГ)
```
Бот: Подписанты из ЗП № 3:
  1. Представитель АО «ОЭК» — Акимов Ю.О., Мастер по ЭРС СВРЭС
  2. Подрядчик АНО «ОЭК Стройтрест» — Буряк А.М., Начальник участка
  3. Субподрядчик ООО «СПЕЦИНЖСТРОЙ» — Щеглов Р.А., Начальник участка
  4. Технадзор АО «ОЭК» — Гайдуков Н.И., Главный специалист ОТН
  Те же или кого меняем?
```

**Правила:**
- ВСЕГДА показывать: номер + организация + ФИО + должность в одной строке.
- Если объект тот же — организации не меняются, меняются только ЛЮДИ.
- Бот НЕ спрашивает "кто заказчик?" в контексте подписантов — это уже известно.

**Если пользователь меняет кого-то:**
```
User: Технадзор — Гайдуков, мастер — Коробков
```
Бот ищет каждого в `memory/people.json`:

Если **найден**:
```
Бот: ✅ Гайдуков Н.И. — есть в базе
  Главный специалист ОТН АО «ОЭК»
  НРС: C-71-259039, распоряжение №01/3349-р от 14.10.2024
✅ Коробков Ю.Н. — есть в базе
  Мастер по ЭРС СВРЭС АО «ОЭК»
```

Если **НЕ найден**:
```
Бот: ❌ Иванов — не найден в базе. Нужны данные:
  1. Полные ФИО (Фамилия И.О.)?
  2. Должность?
  3. НРС-номер и дата? (для технадзора/подрядчика)
  4. Номер и дата приказа/распоряжения?
  Или скиньте PDF распоряжения — я извлеку.
```

**Какие данные нужны по ролям:**

| Роль | ФИО | Должность | НРС | Приказ/Распоряжение |
|------|-----|-----------|-----|---------------------|
| Представитель АО «ОЭК» (мастер) | ✅ | ✅ | ❌ не нужен | ❌ не нужен |
| Подрядчик | ✅ | ✅ | ✅ обязательно | ✅ обязательно |
| Субподрядчик | ✅ | ✅ | ❌ | ✅ приказ |
| Технадзор | ✅ | ✅ | ✅ обязательно | ✅ распоряжение |

#### Шаг 6: Даты + Адрес
```
Бот: Даты работ? Адрес?
```
- Принимает всё в одном сообщении: "начало 10.12.2025, окончание 22.12.2025, Огородный д.11 стр5".
- Парсит даты из любого формата (см. раздел 5).
- Дата акта = дата окончания (по умолчанию). Если отличается — пользователь указывает явно.
- Адрес: если совпадает с предыдущим — можно "тот же" или не указывать.

#### Шаг 7: Параметры ГНБ
```
Бот: Параметры ГНБ:
  - Lплан, Lпроф?
  - Кол-во труб? (по умолчанию 2)
  - Диаметр? (по умолчанию d=225, та же труба)
  Или скиньте паспорт ЗП — извлеку.
```
- Если пользователь кидает PDF паспорта — бот извлекает: Lплан, Lпроф, марку, диаметр, номера паспортов качества, сертификат.
- Кол-во труб по умолчанию = 2 (спросить подтверждение).
- Марка трубы по умолчанию = предыдущий переход (спросить подтверждение).

#### Шаг 8: Сводка
```
Бот: Итого ЗП № 5-5:
  [таблица всех полей]
  Генерирую акты?
```
- ОБЯЗАТЕЛЬНО показать ВСЮ сводку перед генерацией.
- Ждать явное "да" / "ок" / "давай".

#### Шаг 9: Генерация + сохранение
Бот создаёт:
1. **Внутренние акты ОЭК** (11 листов) — из шаблона `Акты ГНБ шаблон v2.xlsx`
2. **АОСР** (3 листа) — из шаблона `АОСР шаблон.xlsx` (TODO)
3. Сохраняет в папку:
```
Работа/[Заказчик]/[Объект]/ЗП [номер]/
  ├── Исполнительная документация/
  │   ├── [номер] Акты ЗП ГНБ [номер].xlsx
  │   └── [номер] АОСР ОЭК-ГНБ [номер].xlsx
  ├── Паспорта на трубу/
  └── Прочее/
```
4. Обновляет `memory/transitions.json`.
5. Сообщает:
```
Бот: ✅ Акты ЗП № 5-5 сгенерированы.
  📁 Сохранено в: Крафт/Марьино/ЗП 5-5/
  📄 Акты ОЭК: 11 листов, 0 ошибок
  📄 АОСР: TODO
```

## 3. Команда /edit — правка существующего перехода

### 3.1. Флоу
```
User: /edit
Бот: Какой переход? [список последних 5]
User: ЗП 5-5
Бот: ЗП 5-5 (Марьино). Что меняем?
User: Адрес — Огородный д.19
Бот: Обновлено:
  Адрес: "г. Москва, Огородный проезд, д. 11, стр. 5"
       → "г. Москва, Огородный проезд, д. 19"
  Перегенерировать акты?
User: Да
Бот: ✅ Акты перегенерированы. Старая версия сохранена как изм 1.
```

### 3.2. Правила правок
- Показать diff ПЕРЕД применением.
- Старую версию НЕ удалять — переименовать с суффиксом `изм N`.
- Номер ревизии — автоинкремент (изм 1, изм 2, изм 3...).
- Записать в `memory/transitions.json` что именно изменилось и когда.

## 4. Обработка входящих документов

### 4.1. Пользователь кидает файл в чат
Бот ОБЯЗАН:
1. Определить тип документа.
2. Если понятно — предложить действие.
3. Если непонятно — спросить.

### 4.2. Типы документов и реакция

| Тип файла | Как определить | Реакция бота |
|-----------|---------------|--------------|
| Паспорт качества на трубу | PDF, "Паспорт качества №...", "Климовский трубный завод" | Извлечь: марка, диаметр, №паспорта, дату. Спросить: "Сохранить в Паспорта на трубу?" |
| Сертификат соответствия | PDF, "СЕРТИФИКАТ СООТВЕТСТВИЯ", "АПТС" | Извлечь номер, срок. Сохранить вместе с паспортами. |
| Распоряжение о технадзоре | PDF, "РАСПОРЯЖЕНИЕ", "О назначении... технического надзора" | Извлечь ФИО, НРС, даты. Обновить `memory/people.json`. |
| Исполнительная схема | PDF/DWG, чертёж с профилем ГНБ | Извлечь Lплан, Lпроф если возможно. Сохранить в папку ЗП. |
| Рукописный акт | JPG/PDF, рукописный текст | OCR → показать распознанное → спросить подтверждение. |
| Фото с объекта | JPG, фото | Спросить: "Что на фото? Сохранить?" |
| Неизвестный документ | Не удалось определить | Спросить: "Что это за документ? Нужно ли его сохранить?" |

### 4.3. Именование файлов при сохранении

**Правило:** файлы ВСЕГДА получают понятные имена на русском.

| Тип | Формат имени | Пример |
|-----|-------------|--------|
| Паспорт качества | `Паспорт качества №[N] [марка].pdf` | `Паспорт качества №13043 ЭЛЕКТРОПАЙП 225.pdf` |
| Сертификат соответствия | `Сертификат соответствия [номер].pdf` | `Сертификат соответствия Н00180-24.pdf` |
| Распоряжение ТН | `Распоряжение ТН №[N] от [дата].pdf` | `Распоряжение ТН №01-3349-р от 14.10.2024.pdf` |
| Исполнительная схема | `Исп.схема ЗП [номер].pdf` | `Исп.схема ЗП 5-5.pdf` |
| Акты ОЭК | `[N] Акты ЗП ГНБ [номер].xlsx` | `5 Акты ЗП ГНБ 5-5.xlsx` |
| АОСР | `[N] АОСР ОЭК-ГНБ [номер].xlsx` | `5 АОСР ОЭК-ГНБ 5-5.xlsx` |
| Ревизия | `[имя] изм [N].xlsx` | `5 Акты ЗП ГНБ 5-5 изм 1.xlsx` |

Если бот **не может определить** тип документа:
```
Бот: Не могу определить тип документа. Что это?
  1. Паспорт качества
  2. Сертификат
  3. Распоряжение
  4. Исп. схема
  5. Другое (опишите)
User: 5 — это письмо о сертификации УКПТ
Бот: Сохраняю как "Письмо о сертификации УКПТ.pdf" в папку ЗП 5-5/Прочее/. Верно?
```

## 5. Правила парсинга данных

### 5.1. Даты
Бот принимает любой формат:
- `10.12.2025` → day=10, month="декабря", year=2025
- `10 декабря 2025` → то же
- `10/12/2025` → то же
- `2025-12-10` → то же

Карта месяцев: январь→января, февраль→февраля, март→марта, апрель→апреля, май→мая, июнь→июня, июль→июля, август→августа, сентябрь→сентября, октябрь→октября, ноябрь→ноября, декабрь→декабря.

Форматирование для внутренних актов: `«{day}» {month_genitive} {year} г.`
Для АОСР: компоненты раздельно (day, month_text, year).

### 5.2. Номер ГНБ
- `"5-5"` → gnb_number=`"ЗП № 5-5"`, gnb_number_short=`"5-5"`
- `"ЗП 5-5"` → то же
- `"ЗП № 5-5"` → то же
- `"3"` → gnb_number=`"ЗП № 3"`, gnb_number_short=`"3"`

### 5.3. Подписанты
- `"Гайдуков"` → поиск по фамилии в people.json
- `"Гайдуков Н.И."` → точный поиск
- `"Коробков вместо Акимова"` → замена sign1
- `"технадзор — Гайдуков, мастер — Коробков"` → парсинг двух замен

### 5.4. Адрес
- `"Огородный д.11 стр5"` → `"г. Москва, Огородный проезд, д. 11, стр. 5"`
- Бот дополняет "г. Москва" если город не указан (по умолчанию — Москва).
- Бот нормализует сокращения: д.→д., стр.→стр., пр.→проезд.

## 7. Самообучение

### 7.1. Что бот запоминает

После КАЖДОГО завершённого `/new` или `/edit` бот обновляет:

1. **`memory/transitions.json`** — история всех переходов:
   ```json
   {
     "transitions": [
       {
         "id": "kraft-marjino-5-5",
         "customer": "Крафт",
         "object": "Марьино",
         "gnb_number": "ЗП № 5-5",
         "created": "2025-12-22",
         "data": { /* полный JSON перехода */ },
         "revisions": [
           { "date": "...", "changes": "адрес изменён", "version": "изм 1" }
         ]
       }
     ]
   }
   ```

2. **`memory/people.json`** — база персонала (см. PERSONNEL_DB.md):
   - Новые люди добавляются автоматически после подтверждения.
   - Если данные обновились (новый приказ) — обновить запись.

3. **`memory/customers.json`** — заказчики и их объекты:
   ```json
   {
     "customers": {
       "kraft": {
         "name": "Крафт",
         "aliases": ["крафт", "kraft"],
         "objects": {
           "marjino": {
             "name": "Марьино",
             "path": "Крафт/Марьино",
             "last_gnb": "ЗП № 5-5"
           }
         }
       }
     }
   }
   ```

4. **`memory/preferences.json`** — предпочтения пользователя:
   ```json
   {
     "default_city": "г. Москва",
     "default_pipe_count": 2,
     "question_order": ["customer", "object", "gnb_number", "base", "signatories", "dates_address", "gnb_params"],
     "typical_responses": {
       "pipe_same": true,
       "sign3_present": true
     }
   }
   ```

### 7.2. Предугадывание ответов

Бот анализирует историю и предлагает наиболее вероятные ответы:

- **Заказчик:** если последние 5 переходов — Крафт, предложить "Крафт" по умолчанию.
- **Подписанты:** если на последних 3 переходах был Гайдуков — предложить его.
- **Труба:** если на объекте всегда d=225, не спрашивать — подтвердить.
- **Адрес:** если у объекта один адрес — подставить и спросить "тот же?".

Пример:
```
Бот: Новый ГНБ-переход. Заказчик — Крафт? (последние 5 переходов)
User: Да
Бот: Объект — Марьино? (последний активный)
User: Да
Бот: Номер перехода? (последний — ЗП № 5-5)
User: 7
```

### 7.3. Адаптация порядка вопросов

Если пользователь систематически отвечает на несколько вопросов одним сообщением, бот запоминает это и в следующий раз объединяет вопросы:

До адаптации:
```
Бот: Даты?
User: 10-22.12.2025, Огородный д.11 стр5
```

После адаптации:
```
Бот: Даты и адрес?
```
