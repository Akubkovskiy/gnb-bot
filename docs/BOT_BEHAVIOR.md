# Модель поведения GNB-Bot — Диалог /new

## Принципы

1. **Минимум вопросов** — если данные наследуются от предыдущего перехода, не переспрашивать.
2. **Группировка** — если объект тот же, организации не меняются. Спрашиваем только про подписантов.
3. **Новые люди** — если подписант неизвестен базе, обязательно запросить все данные (должность, НРС, приказ) или принять документ (PDF распоряжения).
4. **Дефолты** — дата акта = дата окончания, если не сказано иное.

---

## Флоу /new

### Шаг 1: Заказчик
```
Бот: Новый ГНБ-переход. Кто заказчик?
User: Крафт
```
Бот определяет заказчика, смотрит его папку в рабочей директории.

### Шаг 2: Объект
```
Бот: Крафт. Какой объект?
  1. Марьино
  2. [другие объекты из папки]
User: 1
```
Бот показывает объекты из файловой системы заказчика.

### Шаг 3: Основа (предыдущий переход)
```
Бот: Марьино. Беру данные из ЗП № 3 как основу.
```
Бот находит последний/указанный переход и загружает его данные как шаблон.

### Шаг 4: Подписанты (ОДИН вопрос)
```
Бот: Подписанты из ЗП № 3:
  1. Представитель АО «ОЭК» — Акимов Ю.О., Мастер по ЭРС СВРЭС
  2. Подрядчик АНО «ОЭК Стройтрест» — Буряк А.М., Начальник участка
  3. Субподрядчик ООО «СПЕЦИНЖСТРОЙ» — Щеглов Р.А., Начальник участка
  4. Технадзор АО «ОЭК» — Попов А.Д., Главный специалист ОТН
  Те же или меняем?
```

**Важно:** показываем СРАЗУ организацию + должность + ФИО в одной строке.
Объект тот же → организации не меняются → спрашиваем ТОЛЬКО про людей.

### Шаг 4а: Новый подписант (если кого-то меняют)
```
User: Технадзор — Гайдуков Н.И.
Бот: Гайдуков Н.И. — есть в базе ✅
  - Главный специалист ОТН АО «ОЭК»
  - НРС: C-71-259039 от 23.09.2022
  - Распоряжение: №01/3349-р от 14.10.2024
```

Если **нет в базе**:
```
Бот: Гайдуков Н.И. — не найден в базе. Нужны данные:
  1. Должность?
  2. НРС-номер и дата?
  3. Номер и дата распоряжения/приказа?
  (или скиньте PDF распоряжения — я извлеку)
```

### Шаг 5: Даты + Адрес
```
Бот: Обновлённые подписанты:
  [список с пометкой ← изменён]
  Даты работ? Адрес?
```
Можно в одном сообщении — бот парсит всё сразу.

### Шаг 6: Параметры ГНБ
```
Бот: Параметры ГНБ:
  - Lплан, Lпроф?
  - Диаметр?
  - Марка трубы та же (ЭЛЕКТРОПАЙП 225/170...)?
  (или скиньте паспорт ЗП — я извлеку)
```

### Шаг 7: Сводка + подтверждение
```
Бот: [таблица со всеми данными]
  Генерирую акты?
User: Да
```

### Шаг 8: Генерация
Бот создаёт:
- Внутренние акты ОЭК (11 листов)
- АОСР (3 листа) — TODO
- Сохраняет в папку объекта заказчика

## Правила парсинга

### Даты
- "10.12.2025" → day=10, month="декабря", year=2025
- "10 декабря 2025" → то же
- Месяцы: января, февраля, марта, апреля, мая, июня, июля, августа, сентября, октября, ноября, декабря

### Номер ГНБ
- "5-5" → gnb_number="ЗП № 5-5", gnb_number_short="5-5"
- "ЗП 5-5" → то же
- "ЗП № 5-5" → то же

### Подписанты
- "Гайдуков" → поиск в базе по фамилии
- "Гайдуков Н.И." → точный поиск
- "Коробков вместо Акимова" → замена sign1

## Роли подписантов (терминология)

| Роль в боте | Кто это | В актах (B-колонка) |
|------------|---------|---------------------|
| sign1 — Представитель АО «ОЭК» | Мастер по эксплуатации района (СВРЭС, СВЗРЭС и т.д.) | "Представитель АО «ОЭК»" |
| sign2 — Подрядчик | Начальник участка генподрядной организации | "Представитель подрядной организации {name}" |
| sign3 — Субподрядчик (опцион.) | Начальник участка субподрядчика | "Представитель субподрядной организации {name}" |
| tech_supervisor — Технадзор | Главный специалист ОТН АО «ОЭК» | "Представитель технического надзора АО «ОЭК»" |

**Важно:** sign1 — это НЕ заказчик. Это представитель эксплуатирующей организации (мастер РЭС).
