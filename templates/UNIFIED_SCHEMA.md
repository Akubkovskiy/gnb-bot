# Unified JSON Schema — GNB Bot Data Model

Один JSON = один ГНБ-переход. Из этого JSON бот генерирует:
1. **Внутренние акты ОЭК** (11 листов) — `Акты ГНБ шаблон v2.xlsx`
2. **АОСР** (3 листа) — `АОСР шаблон.xlsx`

---

## Иерархия данных

```
Объект (object)
  └── Переход ГНБ (transition)  ← один JSON = один переход
        ├── Идентификация
        ├── Даты
        ├── Организации (с полными реквизитами для АОСР)
        ├── Подписанты (могут меняться от перехода к переходу!)
        ├── Труба и материалы
        └── Параметры ГНБ
```

## Данные, специфичные для каждого файла

### Только внутренние акты:
- `welding_end_date` (B22 — окончание сварки)
- `permits.sro_number/sro_date` (B39, C39)
- `gnb_params.plan_length` (B31)
- `gnb_params.drill_diameter` (F31)
- `gnb_params.configuration` (G31)
- `alt_signatory` (H8 — Панин Д.А.)

### Только АОСР:
- Полные реквизиты организаций (ОГРН, ИНН, адреса, СРО)
- NRS-идентификаторы подписантов
- Номера приказов и распоряжений
- Сертификаты и паспорта на материалы
- Нормативные ссылки (Приказ Минстроя)

## Формулы форматирования (в коде бота)

```typescript
// Дата для внутренних актов
function formatDateInternal(d: DateObj): string {
  return `«${d.day}» ${d.month} ${d.year} г.`;
}

// Подписант для B-колонки внутренних актов
function formatSignatoryDesc(s: Signatory): string {
  return `${s.org_description}, ${s.position}, ${s.full_name}`;
}

// Подписант для C-колонки (подпись в актах)
function formatSignatorySign(s: Signatory): string {
  return `${s.position} __________ ${s.full_name}`;
}
```
