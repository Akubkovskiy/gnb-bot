# Навык: Правки по замечаниям заказчика

## Триггер
Пользователь отправляет замечания текстом, скриншотом или email

## Алгоритм
1. Распознать замечания (текст / OCR скриншота / email)
2. Найти соответствующий файл акта
3. Сформировать список правок
4. Показать diff (было → стало)
5. После подтверждения — внести изменения
6. Сохранить с суффиксом изм[N]
7. Обновить историю ревизий в gnb-transitions.json
8. Отправить файл в Telegram
